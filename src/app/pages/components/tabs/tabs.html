Tabs Component

A flexible and accessible tab component for organizing content into separate views. Switch between different sections with a simple click or keyboard navigation.

Getting Started

Installation

Import the CustomTab component in your Angular component:

        
          import {{ '{' }} CustomTab {{ '}' }} from '@components/custom-tab/custom-tab';
          @Component({{ '{' }}
            selector: 'my-component',
            imports: [CustomTab],
            template: `...`
          {{ '}' }})

Basic Usage

Create a simple tab component with basic configuration:

<custom-tab [tabs]="tabItems" />

Define your tab items in the component:

import {{ '{' }} signal {{ '}' }} from '@angular/core';
      import {{ '{' }} TabItem {{ '}' }} from '@components/custom-tab/custom-tab-model';

      tabItems = signal<TabItem[]>([
        {{ '{' }} id: 1, label: 'Overview', value: 'overview' {{ '}' }},
        {{ '{' }} id: 2, label: 'Details', value: 'details' {{ '}' }},
        {{ '{' }} id: 3, label: 'Settings', value: 'settings' {{ '}' }}
      ]);

Configuration Options

Tab Variants

The component supports multiple visual styles:

  • basic - Simple text-based tabs
  • boxed - Tabs with box-style containers
  • underline - Tabs with bottom border indicator
  • pills - Rounded pill-style tabs

Tab Configuration

tabConfig = signal<TabConfig>({{ '{' }}
        variant: 'underline',
        orientation: 'horizontal',
        selectionMode: 'explicit',
        showIcon: true,
        iconPosition: 'start',
        tabsInLoop: true,
        preserveContent: true
      {{ '}' }});

Component Properties

Inputs

Property Type Default Description
tabs TabItem[] required Array of tab items to display
tabConfigs TabConfig DEFAULT_CONFIGS Configuration object for tab behavior and styling
selectedTab string undefined Two-way binding for selected tab value

TabItem Interface

Property Type Required Description
id any Yes Unique identifier for the tab
label string Yes Display text for the tab
value string Yes Unique value identifier
selected boolean No Initially selected state
icon string No Icon name or path
iconClass string No CSS class for icon styling
disabled boolean No Disabled state of the tab
command () => void No Callback function on tab click

Advanced Usage

Custom Templates

Use custom templates for tab headers and content:

<custom-tab [tabs]="tabItems" [(selectedTab)]="activeTab">
      <ng-template #tabs let-tab="tab">
        <span class="custom-tab-header">
          <i [class]="tab.iconClass"></i>
          {{ '{{' }} tab.label {{ '}}' }}
        </span>
      </ng-template>

  <ng-template #tabsContent let-tab="tab">
    <div class="custom-content">
      <h3>{{ '{{' }} tab.label {{ '}}' }}</h3>
      <p>Custom content for {{ '{{' }} tab.value {{ '}}' }}</p>
    </div>
  </ng-template>
</custom-tab>

Selection Modes

Control how tab selection works:

  • explicit - Requires click to change tabs (default)
  • follow - Changes on keyboard focus

Orientation

Tabs can be displayed horizontally or vertically:

tabConfig = signal<TabConfig>({{ '{' }}
  orientation: 'vertical',
  variant: 'pills'
{{ '}' }});

Accessibility

The component follows ARIA best practices:

  • Built on Angular's ARIA tabs primitives
  • Full keyboard navigation support (Arrow keys, Home, End)
  • Proper ARIA attributes (role, aria-selected, aria-controls)
  • Focus management and visual indicators
  • Screen reader announcements

Keyboard Navigation

Key Action
Arrow Right/Down Navigate to next tab
Arrow Left/Up Navigate to previous tab
Home Navigate to first tab
End Navigate to last tab
Enter/Space Activate focused tab (explicit mode)

Best Practices

  • Keep tab labels short and descriptive (1-2 words)
  • Use 3-7 tabs for optimal usability
  • Don't nest tabs within tabs
  • Use icons to enhance recognition, not replace labels
  • Maintain consistent tab order across page loads
  • Consider using preserveContent: true for forms to prevent data loss
  • Set meaningful default selection using selected property
